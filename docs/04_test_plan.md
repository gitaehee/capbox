# 04. 테스트 계획

## 1. 테스트 목적

본 문서는 CapBox의 주요 기능이 요구사항에 맞게 동작하는지 확인하기 위한 테스트 계획을 정리한 문서이다.

CapBox는 스크린샷 이미지에서 텍스트를 추출하고, 추출된 텍스트를 저장한 뒤, 사용자가 검색어로 다시 찾을 수 있도록 하는 웹 기반 아카이빙 서비스이다.

따라서 테스트는 다음 기능을 중심으로 진행한다.

1. 이미지 업로드
2. OCR 텍스트 추출
3. OCR 결과 수정
4. SQLite 저장
5. 저장 목록 조회
6. 키워드 검색
7. 유사도 검색
8. 하이브리드 검색
9. 예외 상황 처리

---

## 2. 테스트 범위

이번 테스트의 범위는 다음과 같다.

| 구분 | 테스트 대상 |
|---|---|
| 화면 기능 | Streamlit 화면 구성, 입력창, 버튼, 결과 표시 |
| OCR 기능 | 이미지에서 텍스트 추출 여부 |
| 저장 기능 | OCR 결과 저장, 제목 자동 생성, 빈 텍스트 방지 |
| 조회 기능 | 저장된 아카이브 목록 표시 |
| 검색 기능 | 키워드 검색, 유사도 검색, 하이브리드 검색 |
| 예외 처리 | 빈 입력, 검색 결과 없음, OCR 실패 상황 |

## 3. 테스트 환경

| 항목 | 내용 |
|---|---|
| 운영체제 | macOS |
| 언어 | Python |
| 실행 방식 | Streamlit |
| 데이터베이스 | SQLite |
| OCR 라이브러리 | EasyOCR |
| 검색 방식 | 키워드 검색, sentence-transformers 기반 유사도 검색 |
| 실행 명령어 | `streamlit run app.py` |

---

## 4. 테스트 데이터

테스트에는 다음과 같은 유형의 이미지를 사용한다.

| 테스트 이미지 유형 | 설명 |
|---|---|
| 텍스트가 선명한 스크린샷 | OCR이 정상적으로 동작하는지 확인 |
| 한글과 영어가 섞인 스크린샷 | 다국어 OCR 처리 확인 |
| 글자가 작거나 흐린 스크린샷 | OCR 한계 확인 |
| 텍스트가 거의 없는 이미지 | OCR 실패 또는 빈 결과 처리 확인 |
| 상품명/브랜드명이 포함된 스크린샷 | 키워드 검색과 유사도 검색 비교 |
| 과제 안내문처럼 문장이 긴 스크린샷 | 유사도 검색 성능 확인 |

## 7. 테스트 통과 기준

각 테스트 케이스의 통과 기준은 다음과 같다.

1. 기능이 요구사항에 맞게 동작해야 한다.
2. 사용자가 입력 실수를 했을 때 적절한 안내 메시지가 표시되어야 한다.
3. OCR 결과가 일부 부정확하더라도 사용자가 수정할 수 있어야 한다.
4. 저장된 데이터가 목록과 검색 결과에 정상적으로 반영되어야 한다.
5. 검색 결과에는 결과의 근거가 되는 텍스트가 표시되어야 한다.
6. 유사도 검색 결과에는 점수가 표시되어야 한다.

## 8. 테스트 결과 기록 방식

테스트 수행 결과는 별도 문서인 `docs/test_result.md`에 정리한다.

각 테스트 결과는 다음 형식으로 기록한다.

| 테스트 ID | 테스트 내용 | 결과 | 비고 |
|---|---|---|---|
| TC-XXX-01 | 테스트 설명 | 성공/실패/부분 성공 | 발견한 문제 또는 개선 필요 사항 |

---

## 9. 예상되는 품질 이슈

테스트 전 예상되는 품질 이슈는 다음과 같다.

1. OCR이 일부 한글 단어를 잘못 인식할 수 있다.
2. 글자가 작거나 흐린 이미지는 OCR 정확도가 낮을 수 있다.
3. 유사도 검색은 짧은 상품명이나 고유명사 검색에서 부정확할 수 있다.
4. threshold 값에 따라 검색 결과가 너무 많거나 너무 적을 수 있다.
5. 저장된 항목이 많아질 경우 검색 속도가 느려질 수 있다.

---

## 10. 테스트 계획의 의의

본 테스트 계획은 바이브코딩으로 구현한 기능이 실제 요구사항을 만족하는지 확인하기 위한 기준이다.

AI를 활용하면 기능 구현 속도는 빨라질 수 있지만, 생성된 기능이 항상 정확하고 품질이 높은 것은 아니다. 따라서 테스트 케이스를 미리 정의하고, 정상 상황뿐 아니라 예외 상황과 품질 이슈까지 점검하는 과정이 필요하다.

이번 테스트 계획은 이후 테스트 수행 및 품질 관리 단계의 기준으로 활용한다.
