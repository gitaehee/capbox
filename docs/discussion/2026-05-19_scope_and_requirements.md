# 2026-05-19 토의 기록: 범위 조정 및 요구사항 확정

## 1. 오늘의 목표

5월 17일에 선정한 “스크린샷 텍스트 아카이빙 서비스” 아이디어를 바탕으로,
과제 기간 안에 구현 가능한 MVP 범위를 확정하고 기능 요구사항과 비기능 요구사항을 정리한다.

## 2. 추가 검토 내용

초기 아이디어는 사용자가 스크린샷을 찍은 뒤 모바일 앱으로 바로 공유하면,
앱이 이미지 속 텍스트를 추출하여 저장하고,
나중에 사용자가 정확한 문장을 기억하지 못해도 유사도 검색으로 해당 내용을 찾을 수 있도록 하는 서비스였다.

하지만 모바일 공유 기능, 갤러리 접근 권한, 계정 관리, 클라우드 저장까지 포함하면
과제 기간 안에 구현 범위가 과도하다고 판단하였다.

따라서 이번 과제에서는 핵심 가치를 다음과 같이 정의하였다.

> 스크린샷 이미지 속 텍스트를 추출하고, 나중에 검색 가능하게 저장하는 것

## 3. MVP 범위 결정

이번 프로젝트에서는 다음 기능만 구현한다.

1. 사용자가 스크린샷 이미지를 업로드한다.
2. 시스템이 이미지에서 텍스트를 추출한다.
3. 사용자가 OCR 결과를 확인하고 필요하면 수정한다.
4. 수정된 텍스트를 SQLite에 저장한다.
5. 저장된 텍스트 목록을 조회한다.
6. 검색어를 입력하면 저장된 텍스트 중 유사한 항목을 찾아준다.

## 4. 제외하기로 한 기능

다음 기능은 구현 범위에서 제외하고, 향후 확장 기능으로 분류하였다.

| 제외 기능 | 제외 이유 |
|---|---|
| 모바일 앱 공유 기능 | Android/iOS 공유 기능 구현이 필요하여 과제 범위를 초과함 |
| 갤러리 자동 접근 | 권한 처리와 모바일 앱 개발이 필요함 |
| 로그인/회원가입 | 핵심 기능 검증과 직접 관련이 적음 |
| 클라우드 저장 | 개인정보 및 배포 부담이 증가함 |
| 고도화된 OCR 보정 | 시간 대비 구현 부담이 큼 |

## 5. 기술 선택

이번 프로젝트는 빠른 MVP 구현과 화면 캡처의 용이성을 고려하여 Streamlit 기반 웹앱으로 구현한다.

사용 기술은 다음과 같다.

- Python
- Streamlit
- SQLite
- OCR 라이브러리
- sentence-transformers 또는 cosine similarity 기반 유사도 검색

## 6. 오늘의 결정 사항

최종 개발 대상 SW는 “SnapText Archive”로 정하였다.

이 서비스는 스크린샷 이미지를 업로드하면 이미지 속 텍스트를 추출하고,
이를 저장한 뒤 나중에 검색어와 유사한 저장 내용을 찾아주는 웹 기반 아카이빙 서비스이다.
