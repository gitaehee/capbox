# 2026-05-27 토의 기록: 검색 품질 점검 및 개선 방향 정리

## 1. 오늘의 목표

5월 26일까지 구현한 키워드 검색과 유사도 검색 기능을 실제 데이터로 테스트하고,
검색 결과의 품질을 점검한다.

오늘의 목표는 검색 기능이 단순히 동작하는지 확인하는 것을 넘어,
사용자가 기대하는 방식으로 원하는 결과를 찾을 수 있는지 검토하는 것이다.

## 2. 현재 검색 기능 상태

현재 CapBox에는 두 가지 검색 방식이 구현되어 있다.

1. 키워드 검색
2. 유사도 검색

키워드 검색은 사용자가 입력한 검색어가 저장된 아카이브의 제목 또는 본문에 포함되어 있으면 결과를 반환한다.

유사도 검색은 검색어와 저장된 텍스트를 임베딩 벡터로 변환한 뒤, cosine similarity를 계산하여 의미적으로 가까운 결과를 반환한다.

## 3. 테스트 사례

검색어로 “아디다스 컨트리”를 입력하여 유사도 검색을 수행하였다.

검색 결과에는 다음과 같은 항목들이 함께 반환되었다.

| 검색 결과 | 유사도 점수 |
|---|---|
| 자라 옷 | 0.459 |
| 자라 옷 | 0.449 |
| 인스타 릴스에서 본 옷 | 0.391 |

## 4. 발견한 문제

검색어는 “아디다스 컨트리”였지만,
결과에는 “자라 옷”, “인스타 릴스에서 본 옷”처럼 직접적으로 관련성이 낮은 항목도 포함되었다.

이는 유사도 검색 모델이 “아디다스”, “자라”, “옷”, “신발” 등을 모두 패션 또는 상품이라는 넓은 의미 범주로 비슷하게 판단했기 때문으로 보인다.

즉, 유사도 검색은 문장 의미 비교에는 도움이 되지만,
브랜드명, 상품명, 고유명사처럼 정확한 단어 일치가 중요한 검색에서는 기대보다 성능이 낮을 수 있음을 확인하였다.

## 5. 키워드 검색과 유사도 검색의 비교

| 검색 방식 | 장점 | 한계 |
|---|---|---|
| 키워드 검색 | 정확한 단어가 포함된 경우 결과가 명확함 | 표현이 다르면 검색이 어려움 |
| 유사도 검색 | 정확한 단어를 몰라도 의미적으로 비슷한 결과를 찾을 수 있음 | 짧은 상품명이나 고유명사 검색에서는 부정확할 수 있음 |

이번 테스트를 통해 두 검색 방식 중 하나만 사용하는 것보다,
검색 목적에 따라 적절히 조합하는 것이 필요하다는 점을 확인하였다.

## 6. 개선 방향

검색 품질을 개선하기 위해 다음 방향을 고려하였다.

1. 유사도 검색의 기준값을 높인다.
2. 키워드 검색 결과를 우선적으로 보여준다.
3. 키워드 검색 결과가 없을 때 유사도 검색을 보조적으로 사용한다.
4. 검색 결과에 유사도 점수를 표시하여 사용자가 관련성을 판단할 수 있도록 한다.

특히 “아디다스 컨트리”처럼 브랜드명이나 상품명 검색에서는 키워드 검색이 더 적합하고,
“과제 조건”, “보고서 제출 내용”처럼 의미 중심 검색에서는 유사도 검색이 더 적합하다고 판단하였다.

## 7. 오늘의 결론

유사도 검색은 키워드 검색보다 항상 좋은 방식은 아니었다.
검색 대상이 짧은 상품명이나 고유명사일 경우에는 오히려 키워드 검색이 더 정확할 수 있다.

따라서 CapBox의 검색 기능은 키워드 검색과 유사도 검색을 모두 제공하되,
향후에는 키워드 검색을 우선 적용하고 결과가 부족할 때 유사도 검색을 사용하는 하이브리드 검색 방식으로 개선하는 것이 적절하다고 판단하였다.
