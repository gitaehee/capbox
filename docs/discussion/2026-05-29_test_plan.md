# 2026-05-29 토의 기록: 테스트 계획 수립

## 1. 오늘의 목표

오늘은 CapBox의 주요 기능에 대한 테스트 계획을 수립한다.

지금까지 OCR, SQLite 저장, 저장 목록 조회, 키워드 검색, 유사도 검색, 하이브리드 검색 기능을 구현하였다. 따라서 이제는 구현된 기능이 요구사항에 맞게 동작하는지 확인하기 위한 테스트 케이스를 작성한다.

## 2. 테스트 계획이 필요한 이유

바이브코딩을 활용하면 코드 초안을 빠르게 만들 수 있지만, 생성된 코드가 항상 요구사항을 정확히 만족하는 것은 아니다.

특히 OCR과 유사도 검색처럼 입력 데이터에 따라 결과가 달라지는 기능은 직접 테스트를 통해 한계와 품질 문제를 확인해야 한다.

따라서 테스트 계획을 먼저 작성하고, 이후 실제 테스트 결과를 기록하기로 하였다.

## 3. 테스트 범위

이번 테스트에서는 다음 기능을 대상으로 한다.

1. 이미지 업로드
2. OCR 텍스트 추출
3. OCR 결과 수정
4. SQLite 저장
5. 저장 목록 조회
6. 키워드 검색
7. 유사도 검색
8. 하이브리드 검색
9. 예외 상황 처리

## 4. 테스트 케이스 구성

테스트 케이스는 정상 동작만 확인하는 것이 아니라, 다음과 같은 예외 상황도 포함하도록 구성하였다.

- 이미지가 업로드되지 않은 경우
- OCR 결과가 비어 있는 경우
- 저장할 본문이 비어 있는 경우
- 검색어가 비어 있는 경우
- 검색 결과가 없는 경우
- 유사도 검색 결과가 부정확한 경우

## 5. 오늘의 결정 사항

테스트 계획 문서는 `docs/04_test_plan.md`에 작성하고, 실제 테스트 결과는 다음 단계에서 `docs/test_result.md`에 기록하기로 하였다.

테스트 결과는 성공, 실패, 부분 성공으로 구분하여 작성한다.
