# 2026-05-28 토의 기록: 검색 기능 개선

## 1. 오늘의 목표

5월 27일 검색 품질 점검에서 발견한 유사도 검색의 한계를 개선한다.

기존 유사도 검색은 검색어와 직접적인 관련성이 낮은 항목도 함께 반환하는 문제가 있었다.
따라서 오늘은 키워드 검색과 유사도 검색을 조합한 하이브리드 검색 방식을 적용한다.

## 2. 개선 배경

유사도 검색 테스트에서 “아디다스 컨트리”를 검색했을 때,
“자라 옷”, “인스타 릴스에서 본 옷”과 같은 결과가 함께 반환되었다.

이는 임베딩 모델이 브랜드명이나 상품명을 정확히 구분하기보다,
패션 또는 상품이라는 넓은 의미 범주에서 비슷하다고 판단했기 때문으로 보인다.

이 사례를 통해 유사도 검색이 항상 키워드 검색보다 좋은 것은 아니라는 점을 확인하였다.

## 3. 개선 방향

검색 품질을 개선하기 위해 다음 방식을 적용하였다.

1. 키워드 검색 결과를 우선 반환한다.
2. 키워드 검색 결과가 없는 경우에만 유사도 검색을 수행한다.
3. 유사도 검색 기준값을 기존보다 높인다.
4. 검색 결과에 검색 방식과 점수를 함께 표시한다.

## 4. 구현 내용

`search_utils.py`에 `hybrid_search()` 함수를 추가하였다.

하이브리드 검색은 먼저 `keyword_search()`를 실행한다.
검색어가 제목이나 본문에 직접 포함된 결과가 있으면 해당 결과를 반환한다.

만약 키워드 검색 결과가 없으면 `semantic_search()`를 실행하여
검색어와 저장된 텍스트 간 의미적 유사도를 계산한다.

또한 유사도 검색의 threshold를 높여,
관련성이 낮은 결과가 과도하게 반환되지 않도록 조정하였다.

## 5. 검색 방식 비교

| 검색 방식 | 특징 | 적합한 경우 |
|---|---|---|
| 키워드 검색 | 정확한 단어 포함 여부를 확인 | 브랜드명, 상품명, 고유명사 검색 |
| 유사도 검색 | 의미적으로 비슷한 텍스트를 찾음 | 문장, 개념, 설명 검색 |
| 하이브리드 검색 | 키워드 검색 후 필요 시 유사도 검색 수행 | 일반 사용자가 검색 방식을 명확히 알기 어려운 경우 |

## 6. 확인 결과

하이브리드 검색을 적용한 결과,
검색어가 저장된 텍스트에 직접 포함된 경우에는 키워드 검색 결과가 우선 반환되었다.

또한 정확한 키워드가 없는 경우에는 유사도 검색이 실행되어,
의미적으로 비슷한 결과를 찾을 수 있었다.

검색 결과에는 검색 방식과 점수를 함께 표시하여,
사용자가 결과의 관련성을 더 쉽게 판단할 수 있도록 하였다.

## 7. 프로세스 관점의 의미

이번 개선은 단순히 기능을 추가한 것이 아니라,
구현 후 실제 테스트를 통해 품질 문제를 발견하고 이를 개선한 사례이다.

바이브코딩으로 유사도 검색 기능을 빠르게 구현할 수 있었지만,
실제 검색 결과가 사용자의 기대와 다를 수 있음을 확인하였다.
따라서 AI가 생성한 기능도 테스트와 품질 검토를 거쳐야 하며,
필요한 경우 개발자가 기준값 조정이나 검색 전략 변경을 수행해야 한다는 교훈을 얻었다.
