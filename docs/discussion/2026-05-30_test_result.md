# 2026-05-30 토의 기록: 테스트 수행 및 결과 정리

## 1. 오늘의 목표

5월 29일에 작성한 테스트 계획을 바탕으로 CapBox의 주요 기능을 실제로 테스트하고 결과를 정리한다.

오늘은 새 기능을 추가하기보다, 구현된 기능이 요구사항에 맞게 동작하는지 확인하는 것을 목표로 한다.

## 2. 테스트 대상

오늘 테스트한 기능은 다음과 같다.

1. 이미지 업로드
2. OCR 텍스트 추출
3. OCR 결과 수정
4. SQLite 저장
5. 저장 목록 조회
6. 키워드 검색
7. 유사도 검색
8. 하이브리드 검색
9. 예외 상황 처리

## 3. 테스트 결과 요약

전체적으로 CapBox의 핵심 기능은 정상적으로 동작하였다.

이미지 업로드, OCR 결과 수정, 저장, 목록 조회, 키워드 검색, 하이브리드 검색은 성공적으로 동작하였다.

다만 OCR과 유사도 검색은 입력 데이터의 품질과 검색어 형태에 따라 결과가 달라졌기 때문에 부분 성공으로 기록하였다.

## 4. 발견한 문제

테스트 과정에서 다음 문제를 확인하였다.

1. OCR이 일부 한글 단어를 잘못 인식하였다.
2. 유사도 검색은 짧은 상품명 검색에서 부정확한 결과를 반환할 수 있었다.
3. 모델 로딩 시간이 발생하여 유사도 검색이 처음 실행될 때 다소 느렸다.
4. 현재 저장된 항목의 수정 및 삭제 기능은 제공하지 않는다.

## 5. 개선 방향

테스트 결과를 바탕으로 다음 개선 방향을 정리하였다.

1. OCR 결과는 저장 전에 사용자가 수정하도록 유지한다.
2. 키워드 검색과 유사도 검색을 함께 제공한다.
3. 하이브리드 검색을 기본 검색 방식으로 고려한다.
4. 유사도 검색 threshold를 조정하여 관련성이 낮은 결과를 줄인다.
5. 향후 저장 항목 수정 및 삭제 기능을 확장할 수 있다.

## 6. 프로세스 관점의 의미

이번 테스트 수행은 바이브코딩으로 만든 기능이 실제 요구사항을 충족하는지 검증하는 과정이었다.

AI가 생성한 코드는 빠르게 동작하는 기능을 만들 수 있게 해주지만, 그 기능이 실제 사용 상황에서 충분히 정확하고 안정적인지는 별도로 확인해야 한다.

따라서 테스트 단계는 바이브코딩 결과물을 소프트웨어 산출물로 신뢰하기 위한 필수 과정임을 확인하였다.
