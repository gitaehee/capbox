# 2026-05-24 토의 기록: 키워드 검색 기능 구현

## 1. 오늘의 목표

5월 23일에 구현한 SQLite 저장 기능을 바탕으로,
저장된 OCR 텍스트를 다시 찾을 수 있는 검색 기능을 구현한다.

오늘의 목표는 먼저 단순 키워드 검색을 구현하여,
저장된 텍스트나 제목에 검색어가 포함된 경우 해당 아카이브를 반환하도록 하는 것이다.

## 2. 구현 범위

오늘 구현한 범위는 다음과 같다.

1. 검색 기능 모듈 작성
2. 제목 기반 키워드 검색
3. 본문 기반 키워드 검색
4. 검색어 미입력 예외 처리
5. 검색 결과 없음 안내 메시지 처리
6. 검색 결과 목록 화면 표시

유사도 검색은 아직 구현하지 않고,
검색 기능의 기본 흐름을 확인하는 데 집중하였다.

## 3. 설계 반영 내용

5월 20일 설계 문서에서는 검색 기능을 `search_utils.py`로 분리하기로 하였다.

오늘 구현에서는 이 설계를 반영하여,
검색 로직을 `app.py` 안에 직접 작성하지 않고 `search_utils.py`의 `keyword_search()` 함수로 분리하였다.

이를 통해 화면 구성 코드와 검색 처리 코드를 분리할 수 있었고,
이후 유사도 검색 기능을 추가할 때도 `search_utils.py`를 중심으로 확장할 수 있는 구조가 되었다.

## 4. 키워드 검색 방식

오늘 구현한 검색 방식은 사용자가 입력한 검색어가 저장된 아카이브의 제목 또는 본문에 포함되어 있는지 확인하는 방식이다.

예를 들어 저장된 텍스트에 “프로세스에 입각한 바이브코딩의 효과”라는 문장이 있을 때,
사용자가 “바이브코딩”이라고 검색하면 해당 항목이 검색 결과로 반환된다.

## 5. 확인 결과

저장된 아카이브를 대상으로 키워드 검색을 수행한 결과,
제목 또는 본문에 검색어가 포함된 항목이 정상적으로 표시되는 것을 확인하였다.

또한 검색어를 입력하지 않고 검색 버튼을 누르면 경고 메시지가 표시되었고,
검색 결과가 없을 경우 “관련 결과가 없습니다.”라는 안내 메시지가 표시되었다.

## 6. 발견한 한계

키워드 검색은 구현이 단순하고 빠르지만,
사용자가 정확한 단어를 기억해야 한다는 한계가 있었다.

예를 들어 저장된 텍스트에 “바이브코딩 보고서 제출”이라는 문장이 있을 때,
사용자가 “AI 코딩 과제”처럼 의미는 비슷하지만 표현이 다른 검색어를 입력하면 결과를 찾기 어렵다.

또한 OCR 결과가 “젤”을 “절”로 인식하는 것처럼 일부 단어가 잘못 추출된 경우,
정확한 키워드 검색으로는 원하는 결과를 찾지 못할 수 있다.
