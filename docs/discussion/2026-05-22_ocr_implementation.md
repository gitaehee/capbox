# 2026-05-22 토의 기록: OCR 기능 구현

## 1. 오늘의 목표

5월 21일에 구현한 Streamlit 기본 화면과 이미지 업로드 기능을 바탕으로,
업로드한 스크린샷 이미지에서 텍스트를 추출하는 OCR 기능을 구현한다.

오늘의 목표는 이미지 업로드 후 OCR을 실행하고,
추출된 텍스트를 사용자가 확인 및 수정할 수 있도록 화면에 표시하는 것이다.

## 2. 구현 범위

오늘 구현한 범위는 다음과 같다.

1. OCR 라이브러리 의존성 추가
2. OCR 처리 모듈 작성
3. 이미지 업로드 후 OCR 실행 버튼 연결
4. 추출 텍스트를 화면에 표시
5. OCR 결과를 사용자가 수정할 수 있도록 구성
6. OCR 실패 또는 빈 결과에 대한 안내 메시지 추가

DB 저장 기능과 검색 기능은 아직 구현하지 않고,
OCR 기능이 정상적으로 동작하는지 확인하는 데 집중하였다.

## 3. 기술 선택

OCR 라이브러리로 EasyOCR을 사용하였다.

EasyOCR을 선택한 이유는 다음과 같다.

- Python 환경에서 비교적 쉽게 사용할 수 있다.
- 한글과 영어 OCR을 모두 지원한다.
- 별도의 복잡한 서버 설정 없이 로컬에서 실행할 수 있다.
- 과제용 MVP 수준의 OCR 기능 구현에 적합하다.

다만 EasyOCR은 처음 실행 시 모델 로딩 시간이 걸릴 수 있으므로,
Streamlit의 cache 기능을 사용하여 Reader 객체를 한 번만 로드하도록 설계하였다.

## 4. 설계 반영 내용

5월 20일 설계 문서에서는 OCR 기능을 `ocr_utils.py`로 분리하기로 하였다.

오늘 구현에서는 이 설계를 반영하여,
OCR 로직을 `app.py`에 직접 작성하지 않고 `ocr_utils.py`의 `extract_text_from_image()` 함수로 분리하였다.

이를 통해 화면 구성 코드와 OCR 처리 코드를 분리할 수 있었고,
향후 OCR 방식을 변경하더라도 `app.py` 전체를 수정하지 않아도 되는 구조가 되었다.

## 5. 확인 결과

테스트용 스크린샷 이미지를 업로드한 뒤 OCR 실행 버튼을 누르면,
이미지 속 텍스트가 오른쪽 OCR 결과 확인 영역에 표시되는 것을 확인하였다.

또한 추출된 텍스트는 사용자가 직접 수정할 수 있도록 text_area에 표시하였다.

## 6. 발견한 한계

OCR 테스트 결과, 대부분의 텍스트는 추출되었지만 일부 한글 단어에서 오인식이 발생하였다.
예를 들어 “젤”이 “절”로, “살몬”이 “살문”으로, “댓글”이 “대글”로 인식되는 문제가 있었다.

이를 통해 OCR 결과를 그대로 저장하면 데이터 품질이 떨어질 수 있음을 확인하였다.
따라서 본 프로젝트에서는 OCR 결과를 저장하기 전에 사용자가 직접 확인하고 수정할 수 있도록 text_area를 제공하였다.
이는 품질 관리 단계에서 사용자 검증을 포함한 설계가 필요하다는 교훈으로 이어졌다.
