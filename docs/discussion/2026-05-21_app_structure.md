# 2026-05-21 토의 기록: 기본 화면 구조 구현

## 1. 오늘의 목표

5월 20일 설계 문서에서 정의한 화면 흐름을 바탕으로,
Streamlit 앱의 기본 화면 구조를 구현한다.

오늘은 전체 기능을 완성하기보다,
사용자가 이미지를 업로드하고 이후 OCR, 저장, 검색 기능으로 이어지는 화면 흐름을 먼저 확인하는 것을 목표로 한다.

## 2. 구현 범위

오늘 구현 범위는 다음과 같다.

1. Streamlit 앱 기본 설정
2. 서비스 제목과 설명 표시
3. 이미지 업로드 영역 구성
4. 업로드한 이미지 미리보기
5. OCR 결과 확인 영역 배치
6. 저장 목록 영역 배치
7. 검색 영역 배치

OCR, SQLite 저장, 유사도 검색 기능은 아직 구현하지 않고,
각 기능이 들어갈 위치만 먼저 구성하였다.

## 3. 설계 반영 내용

5월 20일 설계 문서에서는 화면을 다음 네 영역으로 나누었다.

- 이미지 업로드 영역
- OCR 결과 확인 영역
- 저장 목록 영역
- 검색 영역

오늘 구현에서는 이 화면 구성을 Streamlit의 `columns`, `file_uploader`, `text_area`, `text_input`, `button` 등을 사용하여 반영하였다.

## 4. 결정 사항

저장 버튼과 검색 버튼은 아직 실제 기능과 연결되지 않았기 때문에 비활성화 상태로 두었다.

이렇게 한 이유는 기능이 구현되지 않은 상태에서 사용자가 버튼을 눌렀을 때 혼동이 생길 수 있기 때문이다.
또한 개발 단계상 UI 구조 확인과 기능 구현을 분리하여 진행하는 것이 더 적절하다고 판단하였다.

## 5. 확인 결과

Streamlit 앱이 정상적으로 실행되었고,
이미지 업로드 시 업로드한 이미지가 화면에 표시되는 것을 확인하였다.

업로드한 이미지에서 텍스트를 추출하고 OCR 결과 확인 영역에 표시되도록 연결한다.
