# 2026-05-17 기록: 과제 주제 선정 및 범위 조정

## 1. 과제 이해

이번 과제는 바이브코딩을 활용하여 소프트웨어를 제작하고,
요구사항 분석부터 품질 관리까지의 소프트웨어 개발 프로세스를 적용한 뒤,
그 효과와 한계를 논술하는 것을 목표로 한다.

## 2. 초기 아이디어

초기 아이디어는 사용자가 나중에 보고 싶은 스크린샷을 저장할 때,
갤러리에 이미지가 계속 쌓이는 문제를 해결하기 위한 서비스였다.

사용자가 스크린샷을 앱에 공유하면,
앱이 이미지에서 텍스트만 추출하여 저장하고,
나중에 사용자가 정확한 문장을 기억하지 못하더라도
유사도 검색을 통해 해당 내용을 다시 찾을 수 있도록 하는 아카이빙 서비스를 구상하였다.

## 3. 범위 조정

초기 아이디어에는 모바일 앱 공유 기능, 갤러리 접근, OCR, 유사도 검색, 계정 관리 등이 포함되어 있었다.
그러나 과제 기간 안에 모든 기능을 구현하기에는 범위가 크다고 판단하였다.

따라서 MVP에서는 다음과 같이 기능 범위를 축소하였다.

- 모바일 앱 공유 기능 → 웹 이미지 업로드 방식으로 대체
- 갤러리 자동 접근 → 사용자가 직접 이미지 업로드
- 계정/로그인 → 로컬 단일 사용자 앱으로 구현
- 벡터DB 검색 → sentence-transformers와 cosine similarity 기반 검색으로 단순화
- OCR 고도화 → OCR 결과를 사용자가 직접 수정할 수 있도록 처리

## 4. 최종 선정 주제

최종적으로 “스크린샷 텍스트 아카이빙 및 유사도 검색 웹앱”을 개발하기로 하였다.

이 서비스는 사용자가 스크린샷 이미지를 업로드하면,
이미지 속 텍스트를 추출하고,
추출된 텍스트를 저장한 뒤,
나중에 검색어와 의미적으로 유사한 저장 내용을 찾아주는 기능을 제공한다.
