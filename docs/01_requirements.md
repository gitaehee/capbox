# 01. 요구사항 분석

## 1. 프로젝트 개요

본 프로젝트는 스크린샷 이미지에 포함된 텍스트를 추출하여 저장하고,
사용자가 나중에 기억나는 단어나 문장으로 해당 내용을 다시 찾을 수 있도록 돕는
웹 기반 스크린샷 텍스트 아카이빙 서비스이다.

서비스명은 “SnapText Archive”로 정하였다.

## 2. 문제 정의

사용자는 나중에 다시 보고 싶은 웹페이지, 게시글, 안내문, 일정, 정보 등을 스크린샷으로 저장하는 경우가 많다.
하지만 시간이 지나면 갤러리에 스크린샷 이미지가 많이 쌓이고,
어떤 이미지에 원하는 정보가 있었는지 찾기 어려워진다.

또한 사용자가 정확한 문장을 기억하지 못하고
“그때 봤던 여행 준비물 글”, “결제 관련 안내”, “과제 제출 조건”처럼
뭉뚱그려서 기억하는 경우도 많다.

따라서 스크린샷 속 텍스트를 추출해 저장하고,
나중에 의미적으로 유사한 검색어로 다시 찾을 수 있는 기능이 필요하다고 판단하였다.

## 3. 개발 목표

본 프로젝트의 목표는 다음과 같다.

1. 사용자가 스크린샷 이미지를 업로드할 수 있도록 한다.
2. 업로드된 이미지에서 텍스트를 추출한다.
3. 추출된 텍스트를 사용자가 확인하고 수정할 수 있도록 한다.
4. 수정된 텍스트를 로컬 DB에 저장한다.
5. 저장된 텍스트 목록을 조회할 수 있도록 한다.
6. 검색어와 유사한 저장 내용을 찾아 보여준다.

## 4. 기능 요구사항

| ID | 기능 요구사항 |
|---|---|
| FR-01 | 사용자는 스크린샷 이미지를 업로드할 수 있다. |
| FR-02 | 시스템은 업로드된 이미지에서 텍스트를 추출할 수 있다. |
| FR-03 | 사용자는 OCR로 추출된 텍스트를 확인하고 수정할 수 있다. |
| FR-04 | 시스템은 사용자가 확정한 텍스트를 SQLite DB에 저장할 수 있다. |
| FR-05 | 사용자는 저장된 텍스트 목록을 조회할 수 있다. |
| FR-06 | 사용자는 검색어를 입력할 수 있다. |
| FR-07 | 시스템은 검색어와 저장된 텍스트 간 유사도를 계산하여 관련 결과를 반환할 수 있다. |
| FR-08 | 시스템은 검색 결과와 함께 유사도 점수를 표시할 수 있다. |

## 5. 비기능 요구사항

| ID | 비기능 요구사항 |
|---|---|
| NFR-01 | 사용자는 복잡한 설정 없이 이미지를 업로드하고 결과를 확인할 수 있어야 한다. |
| NFR-02 | OCR 결과가 부정확할 수 있으므로 사용자가 직접 수정할 수 있어야 한다. |
| NFR-03 | 빈 이미지, 빈 텍스트, 빈 검색어 입력 시 적절한 안내 메시지를 제공해야 한다. |
| NFR-04 | 개인정보가 포함된 스크린샷이 있을 수 있으므로 로컬 DB 기반으로 저장한다. |
| NFR-05 | 검색 결과는 사용자가 이해할 수 있도록 텍스트 일부와 유사도 점수를 함께 보여준다. |
| NFR-06 | 기능별 모듈을 분리하여 유지보수하기 쉽게 구성한다. |

## 6. MVP 범위

이번 과제에서는 다음 범위까지만 구현한다.

- 웹 기반 이미지 업로드
- OCR 텍스트 추출
- OCR 결과 수정
- SQLite 저장
- 저장 목록 조회
- 유사도 기반 검색

## 7. 향후 확장 기능

이번 과제에서는 제외하지만, 실제 서비스로 확장할 경우 다음 기능을 추가할 수 있다.

- 모바일 앱 공유 기능
- 갤러리 자동 접근
- 사용자 계정 및 로그인
- 클라우드 동기화
- 이미지 원본 저장 및 미리보기
- 카테고리/태그 자동 분류
- 검색 결과 하이라이트
