# 테스트 결과

## 1. 테스트 개요

본 문서는 2026년 5월 29일에 작성한 테스트 계획을 바탕으로 CapBox의 주요 기능을 실제로 수행한 결과를 정리한 문서이다.

테스트 대상 기능은 이미지 업로드, OCR 텍스트 추출, OCR 결과 수정, SQLite 저장, 저장 목록 조회, 키워드 검색, 유사도 검색, 하이브리드 검색, 예외 상황 처리이다.

테스트 중 다음과 같은 OCR 오인식이 확인되었다.

| 원본 | OCR 결과 |

이 사례를 통해 OCR 결과를 그대로 저장하는 것은 데이터 품질 측면에서 위험할 수 있음을 확인하였다. 따라서 사용자가 OCR 결과를 직접 확인하고 수정할 수 있도록 한 설계가 필요하다는 점을 확인하였다.

#### 유사도 검색 한계 사례

검색어로 “아디다스 컨트리”를 입력했을 때, “자라 옷”, “인스타 릴스에서 본 옷”처럼 넓은 패션 범주의 결과가 함께 반환되는 문제가 있었다.

이는 유사도 검색 모델이 짧은 상품명이나 브랜드명을 정확히 구분하기보다, 패션/상품이라는 넓은 의미 범주에서 유사하게 판단했기 때문으로 보인다.

| TC-HYBRID-02 | 정확한 키워드가 없는 경우 | 성공 | 유사도 검색으로 보조 검색 수행 |

## 6. 발견한 문제점

테스트 결과 다음과 같은 문제점을 발견하였다.

1. OCR은 이미지 품질에 따라 일부 한글 단어를 잘못 인식한다.
2. 키워드 검색은 정확한 단어가 포함되어야만 결과를 찾을 수 있다.
3. 유사도 검색은 짧은 상품명이나 고유명사 검색에서 부정확한 결과를 반환할 수 있다.
4. 유사도 검색은 모델 로딩 시간이 발생한다.
5. 저장된 항목을 수정하거나 삭제하는 기능은 아직 없다.

---

## 7. 개선 방향

발견한 문제를 바탕으로 다음 개선 방향을 정리하였다.

1. OCR 결과를 저장 전 사용자가 수정할 수 있도록 유지한다.
2. 키워드 검색과 유사도 검색을 함께 제공한다.
3. 하이브리드 검색을 기본 검색 방식으로 사용하는 것을 고려한다.
4. 유사도 검색 threshold를 조정하여 관련성이 낮은 결과를 줄인다.
5. 향후 저장 항목 수정 및 삭제 기능을 추가할 수 있다.

---

## 8. 테스트 수행의 의의

이번 테스트를 통해 바이브코딩으로 구현한 기능이 실제 사용 상황에서 어떻게 동작하는지 확인할 수 있었다.

AI를 활용해 기능 구현 속도는 빨라졌지만, OCR 품질이나 유사도 검색 결과처럼 실제 데이터에 따라 달라지는 부분은 반드시 테스트를 통해 검증해야 했다.

특히 유사도 검색은 구현 자체는 빠르게 가능했지만, 검색 품질이 항상 기대한 수준으로 나오지는 않았다. 따라서 기능 구현 이후 테스트와 품질 관리가 필수적이라는 점을 확인하였다.
